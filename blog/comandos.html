COMANDOS EN TERMINAL

Dentro de la terminal se usan diferentes mensajes que provocan una respuesta un comportamiento especifico.

A continuación se desglosarán algunos de los más importantes:

Comandos básicos

Coamndo Función
clear Limpia la terminal
history Muestra el historial de comandos usados en la terminal
man {comando} Muestra el manual del comando especificado
f Forzar
alias {nombre='comando'} Abreviar comandos

Navegación de archivos

Coamndo Función
pwd Muestra la ruta actual (donde estoy ahorita)
ls Lista el contenido del directorio actual (archivos y carpetas donde estoy ahorita)
ls -l Lista completa de los atributos de todos los archivos que tengo en la carpeta donde estoy
ls -a Muestra los archivos ocultos
ls -la Lista completa de los archivos, incluyendo los archivos ocultos
cd {directorio} Cambia al directorio especificado
cd .. Cambia al directorio anterior
cd Cambia al directorio

Manipulación de archivos y carpetas

Coamndo Función
touch {nombre} Crear un archivo
mkdir {nombre} Crea una carpeta
code . Crea un archivo con visual studio code
rm {archivo} Eliminar un archivo especfico
rm -r {carpeta} Eliminar una carpeta con todo y su contenido
rm {1..10}.{tipo} Eliminar todos los archivos del tipo especificado del 1 al 10
rm *.{tipo} Eliminar todos los archivos del tipo especificado
rm num*.{tipo} Eliminar los archivos que empiezan con ese número y terminan con .tipo especificado
rm num*letra.{tipo} Eliminar los archivos que empiezan con ese número contienen cualquier cosa, luego una letra especificada y terminan con .tipo especificado
mv {origen destino} Mueve archivos (archivo voy a mover y a donde)
mv {nombre} Renombra el archivo o carpeta
cp {origen destino} Agregar el archivo que quieres copiar y a donde lo quieres copiar

Comandos de git (básicos)

Coamndo Función
git init Inicializar un repositorio Git en la carpeta actual
git status Mostrar los archivos modificados y el estado del repo
git add {archivo} Añadir el archivo al área de preparación (staging)
git add . Añadir todos los archivos modificados al área de preparación (staging)
git commit -m "comentario" Registra los cambios en el historial del repositorio local
git log Mostrar el historial de commits
git diff Mostrar los cambios hechos antes de un commit
git reset {archivo} Quitar el archivo del área de preparación (staging)
git clone {SSH/HTTPS} Clonar un repositorio remoto en la computadora
git restore {archivo} Descartar los cambios en un archivo especifico

Comandos de git (Repositorios remotos)

Coamndo Función
git push origin main Envía tus commits al repositorio remoto
git pull origin main Traer los últimos cambios desde el remoto
git pull Jala algo del repositorio remoto al local
git fetch --prune Recuperar ramas eliminadas remotamente
git stash Descargar cambios (no los aplica)

Comandos de git (Ramas)

Coamndo Función
git branch Muestra todas las ramas
git branch -a Verificar las ramas que tenemos
git branch -d {rama} Borrar la rama
git checkout {branch to move to} Regresar a la rama principal o cambiar de rama
git checkout -b {carpeta} Cambiar entre ramas existentes, el parámetro -b, el comando crea la rama si no existe y luego cambia a ella
git switch {rama} Cambiar rama
git merge {rama} Unir una rama con la rama actual
